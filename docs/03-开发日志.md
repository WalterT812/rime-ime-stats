## 2026-06-13 v1.2 代码审查全面优化 + 正式建仓（用户：以 C# 版为主，全部优化）

用户明确今后**主要用 C# 版**。本轮先建正式 git 仓库保住历史，再按代码审查清单把两版的隐患逐条修掉，C# 版重点打磨。

**git 正式建仓（最优先）**
- 此前为绕开 Linux 沙盒挂载盘不支持 git 原子操作，仓库只以 `ime-repo.bundle` 形式存在，且 bundle 停在 v0.9——v1.1 的关键修复只躺在工作目录，**无任何备份**。本轮在原生 Windows 下 `git init`，从 bundle `fetch` 恢复 v0.9 历史后补提交 v1.1，再继续 v1.2。`core.autocrlf=false` 固定 LF，避免全文件 CRLF 翻动
- 收尾会重新生成 `ime-repo.bundle`（含完整 v1.2 历史），今后每次提交后同步刷新

**C# 版（主力，csharp/Program.cs）**
- **快捷键不再误计英文**：原来只看虚拟键 A–Z，导致 Ctrl+C / Ctrl+V 等被算进"英文字符"。新增 `GetAsyncKeyState` 在钩子回调内取 Ctrl/Alt/Win 状态随键入队，Work 线程据此排除组合键，与 Python 版（组合键 `char` 为 None 不计）口径对齐
- **GDI 句柄泄漏修复**：`MakeIcon` 用 `Icon.FromHandle(bmp.GetHicon())` 每次暂停/恢复泄漏一个 HICON。改为 Clone 出托管副本后立即 `DestroyIcon`，并在切换托盘图标时 Dispose 旧图标
- **日志读取不再漏字**：原 `Offset = fs.Length` 在读取与赋值间若 Rime 追加新行会被跳过。改为按字节读入、只消费到最后一个 `\n`、`Offset += 已消费字节`，半行留到下次——与 Python 同一套"按字节推进"逻辑
- **CJK 口径对齐**：`CountCjk` 补扩展 B 区（0x20000–0x2A6DF）并正确处理代理对，生僻字不再漏算
- **轮转竞态修复**：把"追平字数→校验词频→改名归档"全部移进 `_reader.Lock` 内（新增 `PollNoLock`），消除追平与归档之间 Rime 写入丢字的窗口
- **周报长期不重启也能生成**：原来只在启动时查一次，连挂数周不会出周报。改为落盘线程里约每小时调一次 `WeeklyReport`（meta 标记防重复）
- **共用 config.json**：新增 `LoadConfig` 读 `stats-app\config.json` 的 `skip_processes`/`flush_interval_sec`，与 Python 版同一份配置，落盘间隔/跳过进程两版一致

**Python 版（ime_stats.py / word_worker.py）同步**
- `CommitLogReader._poll`、`word_worker` 读日志同样改为"按字节读、消费到末个换行、offset 按字节推进"，与 C# 完全一致，杜绝半行丢字
- `maybe_rotate_log` 追平+归档全程持 `reader.lock`，消除轮转竞态
- 周报检查移入 flusher 循环约每小时一次（跨周即生成）；落盘/轮转/周报周期改用 `3600 // flush_interval_sec` 计算，不再写死 120

**清理**
- 删除迁移前的残留库：`stats-app\stats.db*`、`csharp\publish\stats.db*`（live 库在 `%APPDATA%\IMEStats`，未动）

**实机切换中发现并修复的真 bug：C# 单文件缺原生库启动即崩**
- 切到 C# 版实测时，从 `stats-app\IMEStatsSharp.exe` 启动**秒退**。根因：`PublishSingleFile=true` 默认**不嵌入原生库**，`e_sqlite3.dll`（SQLite 原生，1.6MB）独立躺在 publish 目录旁；而 `build_csharp.bat` 只把 exe 复制到 `stats-app\`，导致那里跑时 `SqliteConnection.Open()` 找不到原生库抛异常，WinExe 无控制台、静默崩溃。
- 之所以一直没暴露：用户此前是直接从 `csharp\publish\`（dll 在旁）跑的，才"看着正常"。
- **修复**：csproj 加 `<IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>`，原生库嵌入单文件、运行时自解压。重新发布后 exe 从 467KB→2.1MB，publish 目录不再有独立 dll，复制到任意位置都能跑。
