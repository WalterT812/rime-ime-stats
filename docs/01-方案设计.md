# 方案设计

## 目标

做一个属于自己的中英文输入法，要求：输入体验接近微软拼音；统计按键次数、中文字数、英文字数；游戏中使用不受影响。

## 技术路线：为什么基于 Rime 而不是从零写

从零写一个 Windows TSF 输入法，框架部分（COM 组件、候选窗、焦点管理）约需数月，而真正决定"好不好用"的是词库和智能整句算法——微软拼音背后是海量语料训练的语言模型，个人短期无法追平。

Rime（中州韵）是成熟的开源输入法引擎，Windows 前端"小狼毫"已解决 TSF 集成、游戏兼容等所有底层问题。搭配社区维护的**雾凇拼音**（rime-ice）方案——百万级词库、整句输入、词频调整——体验公认接近商业输入法。我们在其上做两件事：个性化配置 + 统计系统。

## 统计系统架构

```
┌─ Rime 引擎 ──────────────┐      ┌─ 统计托盘程序 (Python) ─────────┐
│ commit_logger.lua        │      │ 1. pynput 低级键盘钩子 → 按键数  │
│  监听上屏事件             │ 写入  │ 2. 增量读取 commit_log.txt      │
│  → commit_log.txt        │─────→│    → 统计 CJK 字符 = 中文字数    │
└──────────────────────────┘      │ 3. 英文模式下的字母 → 英文字/词  │
                                  │ 4. SQLite 落盘，托盘面板展示     │
                                  └──────────────────────────────────┘
```

### 三个指标的口径

| 指标 | 来源 | 说明 |
|------|------|------|
| 按键次数 | 全局键盘钩子 | 所有按下事件，任何程序中都计 |
| 中文字数 | Rime 上屏日志 | 精确：只统计真正上屏的 CJK 字符，拼音编码不算 |
| 英文字符/单词 | 键盘钩子 + IME 模式检测 | 英文模式下敲的字母计字符，遇空格/标点结算单词 |

### IME 模式检测

通过 `ImmGetDefaultIMEWnd` + `WM_IME_CONTROL(IMC_GETCONVERSIONMODE)` 查询前台窗口中/英状态，避免把拼音编码误计为英文。已知局限：个别纯 TSF 场景查询可能失败，此时字母按英文计，会轻微高估英文数（中文字数不受影响，因为来源是上屏日志）。后续可改进。

## 游戏兼容设计

三层保障：

1. **app_options 自动英文**（weasel.custom.yaml）：游戏进程启动即处于英文直通模式，按键不进中文引擎、不弹候选框、零额外延迟。聊天时按左 Shift 临时切中文。
2. **被动钩子**：统计程序用 `WH_KEYBOARD_LL` 低级钩子，只监听不修改不注入，与各类按键统计软件同原理，主流反作弊均容忍。
3. **逃生通道**：托盘菜单可一键"暂停统计"（完全停掉钩子）；config.json 的 `skip_processes` 可对指定进程跳过计数。

## 已知取舍

- 雾凇拼音词库需联网下载一次（脚本自动完成）
- 统计程序用 Python 是为了快速迭代；如需更低内存占用，后续可用 C# 重写（接口不变：读同一个 commit_log.txt 和 stats.db）
